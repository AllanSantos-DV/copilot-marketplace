voice-chat

v1.2.2

O voice-chat transforma o terminal numa conversa: você fala, o Whisper (ou o motor único vox-engine) transcreve, o agente responde e uma voz local em pt-BR lê o resumo de volta. Nada sai da sua máquina.

⌁ copilot plugin install voice-chat@copilot-marketplace
código↗

  • Fala → texto, local

    Grava o microfone e transcreve com Whisper on-device — sem nuvem, sem chave de API.

  • Resposta falada

    A cada turno injeta a instrução do resumo 🔊 e sintetiza áudio em português no Windows.

  • vox-engine ou fallback

    Usa o motor único vox-engine quando existe (STT compartilhado, melhor qualidade) e cai para o Whisper local quando não.

#O que é

O voice-chat é uma canvas extension do GitHub Copilot CLI: abre um painel de voz no app e liga o ciclo fala → agente → fala. Você segura para falar, ele transcreve localmente, manda o texto ao agente e devolve um resumo em áudio na sua língua.

É pensado para acompanhar sessões longas sem tirar os olhos do que importa: o áudio de saída é um resumo falado, curto e natural, não a leitura crua da resposta inteira.

i

Como registra um canvas, depois de instalar é preciso reiniciar o app uma vez para o painel aparecer (o canvas-sync cuida do espelhamento).

#Como usar

  1. Instale o plugin

    Pelo marketplace, com o comando da seção Instalar abaixo.

  2. Reinicie o app

    Uma vez, para o canvas de voz ser descoberto.

  3. Abra o painel Voz

    O canvas mostra o microfone e o estado (ouvindo, transcrevendo, falando).

  4. Segure para falar

    Fale sua pergunta; ao soltar, o Whisper transcreve e o agente responde com um resumo em áudio.

i

Na primeira fala o modelo de voz pode levar alguns segundos para aquecer; depois fica instantâneo.

#Como funciona

  • Captura — grava o microfone enquanto você segura o botão do painel.
  • STT — transcreve com o vox-engine (se presente) ou com o Whisper local como fallback.
  • Agente — envia o texto transcrito como um turno normal do Copilot.
  • TTS — sintetiza um resumo falado em pt-BR pela voz do Windows (tts.ps1).

O worker de voz (voice_worker.py) roda apartado e é reusado por outros plugins da vitrine que precisam de voz — por isso o STT é compartilhado quando o vox-engine está disponível.

#Instalar

Registre a vitrine uma vez (se ainda não fez):

⌁ copilot plugin marketplace add AllanSantos-DV/copilot-marketplace

Instale o voice-chat:

⌁ copilot plugin install voice-chat@copilot-marketplace

Atualize quando quiser (ou todos com --all):

⌁ copilot plugin update voice-chat
!

Este plugin registra um canvas (painel no app). Depois de instalar, reinicie o app uma vez: o canvas-sync espelha a extensão para ~/.copilot/extensions/ e o app a descobre no próximo boot.

#Perguntas frequentes

Precisa de internet?

Não para a voz: transcrição (Whisper) e síntese rodam localmente. Só o agente do Copilot usa a rede, como de costume.

Funciona no Linux ou macOS?

O núcleo de STT é portável, mas a fala de saída usa a voz do Windows (tts.ps1). Hoje o foco é Windows.
