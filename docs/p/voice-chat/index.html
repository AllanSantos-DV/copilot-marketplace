voice-chat

v1.3.2

O voice-chat transforma o terminal numa conversa: você fala, o motor local vox-engine transcreve, o agente responde e uma voz neural em pt-BR lê o resumo de volta. Fala e voz — STT e TTS — passam pelo mesmo e único motor, na sua máquina, e a extensão cuida dele sozinha: instala se faltar, atualiza se estiver velho e usa quando está em dia. Nenhum áudio sai do computador.

⌁ copilot plugin install voice-chat@copilot-marketplace
código↗

  • Um motor, STT + TTS

    Transcrição e voz passam pelo mesmo vox-engine local — um daemon compartilhado por named pipe. Sem nuvem, sem chave de API.

  • Se instala e se atualiza sozinho

    No arranque a extensão baixa o motor se faltar, atualiza para a última versão se estiver velho e só conecta quando já está em dia. Você não roda nada à mão.

  • O motor é o único caminho

    STT e TTS existem só pelo vox-engine. Não há modo alternativo nem chave para desligá-lo — se o motor não sobe, o erro aparece alto na tela, nunca em silêncio.

  • Windows, ponta a ponta

    O motor conversa por named pipe e é provisionado por um instalador .ps1 — por isso o voice-chat roda no Windows 10/11.

#O que é

O voice-chat é uma canvas extension do GitHub Copilot: abre o painel Voz no app e liga o ciclo fala → agente → fala. Você segura para falar, o motor transcreve localmente, o texto vira um turno normal do Copilot e a resposta volta como um resumo falado, curto e natural, na sua língua.

Fala e voz passam pelo mesmo motor: STT (fala→texto) e TTS (texto→voz) rodam no vox-engine local, um daemon compartilhado com outros plugins da vitrine (como o copilot-mobile). É o único caminho de áudio — a extensão cuida do motor, e o motor cuida da voz.

i

Como registra um canvas, depois de instalar é preciso reiniciar o app uma vez para o painel aparecer (o canvas-sync cuida do espelhamento).

#Como usar

  1. Instale o plugin

    Pelo marketplace, com o comando da seção Instalar abaixo.

  2. Reinicie o app

    Uma vez, para o canvas de voz ser descoberto.

  3. Abra o painel Voz

    Na primeira vez, a extensão instala o motor e baixa os modelos — pode levar alguns minutos; a tela mostra o progresso.

  4. Segure para falar

    Clique no microfone para gravar, clique de novo para enviar. O motor transcreve e o agente responde com áudio.

  5. Mãos livres (opcional)

    Ligue a wake word nas configurações e diga a frase de ativação ("escuta jarvis") para começar sem clicar.

i

Depois da 1ª vez o painel abre em segundos — o motor reusa os modelos e a voz já baixados.

#O motor único (vox-engine)

Toda a fala passa pelo vox-engine: um motor local de STT + TTS que roda como daemon e é compartilhado entre apps por um named pipe. A extensão cuida do ciclo de vida dele no arranque, sem você configurar nada:

  • Não instalado → baixa o instalador da release pública, instala em %LOCALAPPDATA%\vox-engine e sobe o daemon.
  • Desatualizado → atualiza para a última versão antes de subir (best-effort: se falhar, segue no que já funciona).
  • Em dia → só conecta e usa. Se o daemon já está no ar (por exemplo, subiu para o copilot-mobile), ela não interrompe — apenas se liga.
i

Instalações concorrentes são serializadas por um lock entre processos — dois apps subindo o motor ao mesmo tempo não corrompem o venv.

!

O motor é o único caminho. Não existe modo alternativo nem variável para desligá-lo: se o vox-engine não estiver disponível, o erro aparece alto na interface, nunca em silêncio.

#Como funciona

  • Captura — grava o microfone no painel (push-to-talk ou wake word).
  • STT — o voice_worker.py manda o áudio ao motor pelo named pipe e recebe o texto.
  • Agente — o texto transcrito entra como um turno normal do Copilot.
  • TTS — o mesmo motor sintetiza um resumo falado em pt-BR, que toca no canvas.

Por ser um daemon compartilhado, o motor sobe uma vez e é reusado — o STT e o TTS ficam consistentes entre o voice-chat e outros plugins que também falam.

#Instalar

Registre a vitrine uma vez (se ainda não fez):

⌁ copilot plugin marketplace add AllanSantos-DV/copilot-marketplace

Instale o voice-chat:

⌁ copilot plugin install voice-chat@copilot-marketplace

Atualize quando quiser (ou todos com --all):

⌁ copilot plugin update voice-chat
!

Este plugin registra um canvas (painel no app). Depois de instalar, reinicie o app uma vez: o canvas-sync espelha a extensão para ~/.copilot/extensions/ e o app a descobre no próximo boot.

#Estrutura

Runtime vendado em plugins/voice-chat/ — vem do repositório de origem, então não se edita à mão aqui. É isto que o copilot plugin install entrega:

⌁ plugins/voice-chat/
  • extension.mjs Servidor Node do canvas: roteia a voz, orquestra STT/TTS pelo motor, SSE e o auto-update da extensão. 88 KB
  • iframe.html Interface do canvas de voz (push-to-talk e estado do motor). 55 KB
  • plugin.json Metadados do plugin + marcador de canvas (extensions). 632 B
  • requirements.txt Dependências Python do worker de voz — o worker instala o que faltar no arranque. 308 B
  • voice_worker.py Worker Python: ponte para o motor único (STT+TTS) pelo named pipe, com VAD e wake word. 111 KB

#Perguntas frequentes

Precisa de internet?

Só na primeira vez, para baixar o motor e os modelos. Depois STT e TTS rodam 100% local — nenhum áudio sai da máquina. O agente do Copilot usa a rede como de costume.

Preciso instalar o vox-engine na mão?

Não. A extensão instala, atualiza e sobe o motor sozinha no arranque; você só abre o painel Voz.

E se o motor não subir?

O erro aparece alto na interface, nunca em silêncio. Confira o Python 3.11+ no PATH e os logs em %LOCALAPPDATA%\vox-engine\logs, feche o app e abra de novo para a extensão tentar provisionar o motor outra vez.

Atualizar não funciona na rede da empresa?

Em redes que reassinam o HTTPS ("SSL assinado", proxy corporativo com CA própria), o download nativo do Node falha porque o pacote de CAs dele não conhece a CA da empresa. Quando isso acontece, a extensão baixa a atualização sozinha pelo curl.exe do Windows (Schannel), que confia no trust store da máquina — onde o TI já instalou a CA corporativa. É automático, sem configurar nada e sem desligar a verificação TLS (nada de --insecure); o sha256 de cada arquivo continua conferido.

Funciona no Linux ou macOS?

Não. O voice-chat é Windows-only: o motor conversa por named pipe e é provisionado por um instalador .ps1, específicos do Windows. Não há versão para outros sistemas.
