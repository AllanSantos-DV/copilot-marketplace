mcp-bridge

v0.1.0

Onde o MCP é bloqueado por política mas extensões são permitidas, o mcp-bridge reabre a porta: conecta a servidores MCP (stdio, http, sse — com login OAuth) e expõe as tools, prompts e resources deles nativamente ao agente, com resiliência, auditoria e um painel de saúde. Devolve os braços e as pernas da IA.

⌁ copilot plugin install mcp-bridge@copilot-marketplace

    Reconecta o agente a servidores MCP usando o canal de extensões, que a política permite.

  • Resiliência e auditoria

#O que é

O mcp-bridge é uma extensão que fala Model Context Protocol por baixo e reapresenta cada servidor MCP como capacidade nativa do agente do Copilot. É a saída para ambientes onde o MCP direto está barrado por política, mas extensões são liberadas.

Ele não força nada: usa o canal que já é permitido para reconectar o agente a servidores que você controla, e registra tudo para auditoria.

i

Registra um canvas (painel de saúde) e usa hooks; reinicie o app uma vez após instalar.

#Como usar

  1. Instale o plugin

    Pelo marketplace (comando na seção Instalar).

  2. Reinicie o app

    Uma vez, para os hooks e o painel entrarem.

  3. Configure os servidores

    Aponte para seus servidores MCP (stdio, http ou sse) e, se preciso, faça o login OAuth.

  4. Use as capacidades

    As tools, prompts e resources dos servidores passam a aparecer nativamente para o agente.

  5. Acompanhe a saúde

    O painel mostra conexões, reconexões e a auditoria das chamadas.

i

Conexões caídas são retomadas automaticamente; o painel deixa claro o que está de pé e o que falhou.

#Transportes e segurança

  • stdio — processos MCP locais.
  • http e sse — servidores MCP remotos.
  • OAuth — login quando o servidor exige.
  • Auditoria — registro das tools chamadas, para rastreabilidade.

Pensado para uso enterprise: você mantém o controle dos servidores e a visibilidade do que o agente acessa.

#Instalar

Registre a vitrine uma vez (se ainda não fez):

⌁ copilot plugin marketplace add AllanSantos-DV/copilot-marketplace

Instale o mcp-bridge:

⌁ copilot plugin install mcp-bridge@copilot-marketplace

Atualize quando quiser (ou todos com --all):

⌁ copilot plugin update mcp-bridge
!

Este plugin registra um canvas (painel no app). Depois de instalar, reinicie o app uma vez: o canvas-sync espelha a extensão para ~/.copilot/extensions/ e o app a descobre no próximo boot.

#Estrutura

Runtime vendado em plugins/mcp-bridge/ — vem do repositório de origem, então não se edita à mão aqui. É isto que o copilot plugin install entrega:

⌁ plugins/mcp-bridge/
  • boot.mjs Bootstrap de SessionStart: baixa/garante o canvas-sync na máquina. 1.1 KB
  • extension.mjs Extensão in-app: hooks e tools da sessão viva do agente. 751 KB
  • hooks.json Hooks de ciclo de vida (SessionStart e afins). 178 B
  • plugin.json Metadados do plugin + marcadores (extensions / hooks). 823 B
  • sync.mjs Espelha canvas extensions instaladas para ~/.copilot/extensions. 9.4 KB
