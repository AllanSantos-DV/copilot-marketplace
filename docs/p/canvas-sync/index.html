canvas-sync

v0.3.0

O app GUI só carrega canvases de ~/.copilot/extensions/, mas instalar pelo marketplace cai em installed-plugins/. O canvas-sync fecha essa lacuna: num hook de SessionStart, espelha os plugins marcados como canvas para a pasta certa. Idempotente e seguro — você só escolhe o plugin.

⌁ copilot plugin install canvas-sync@copilot-marketplace
código↗

  • Espelha automaticamente

    No início da sessão, copia os plugins com marcador extensions para ~/.copilot/extensions/.

  • Idempotente e seguro

    Usa um stamp e nunca sobrescreve uma pasta sem stamp — sua cópia de dev fica intacta.

  • Zero passo manual

    Cada plugin canvas traz um boot.mjs que baixa o canvas-sync sozinho quando ele falta.

#O que é

O canvas-sync é a infra da própria vitrine. Não é um canvas nem tem UI: é um hook de SessionStart que resolve um detalhe do app — canvases só carregam de uma pasta específica, e o marketplace instala em outra.

Ele faz a ponte entre os dois lugares, para que qualquer plugin canvas instalado pelo marketplace simplesmente funcione depois de um reinício.

i

Só usa hooks (não registra canvas). É a peça que os outros plugins canvas dependem por baixo.

#Por que existe

O app GUI descobre canvases apenas em ~/.copilot/extensions/. Instalar um plugin pelo marketplace cai em ~/.copilot/installed-plugins/ e, sem ajuda, não roda como canvas.

  • Sem o canvas-sync, cada plugin canvas exigiria cópia manual para extensions/.
  • Com ele, o espelhamento acontece sozinho no início da sessão.
  • O marcador oficial extensions no plugin.json é o que diz o que espelhar.

#Como funciona

  1. Roda no SessionStart

    O hook executa node sync.mjs no início de cada sessão.

  2. Lê os plugins ativos

    Filtra em settings.json → enabledPlugins os que têm o marcador extensions.

  3. Espelha para extensions/

    Copia installed-plugins/<mp>/<plugin>/ → extensions/<plugin>/.

  4. Marca com stamp

    Grava .canvas-sync.json; é idempotente e nunca sobrescreve uma pasta sem stamp.

!

Após instalar ou atualizar plugins, reinicie o app uma vez: o hook popula extensions/ no início da sessão e o app descobre os canvases no próximo boot.

#Instalar

Registre a vitrine uma vez (se ainda não fez):

⌁ copilot plugin marketplace add AllanSantos-DV/copilot-marketplace

Instale o canvas-sync:

⌁ copilot plugin install canvas-sync@copilot-marketplace

Atualize quando quiser (ou todos com --all):

⌁ copilot plugin update canvas-sync

#Estrutura

#Perguntas frequentes

Preciso instalar isto à mão?

Normalmente não: os plugins canvas baixam o canvas-sync sozinhos via boot.mjs. Você pode instalá-lo junto se quiser garantir.

Por que preciso reiniciar o app?

O hook popula extensions/ no início da sessão; o app descobre os canvases no próximo boot.
