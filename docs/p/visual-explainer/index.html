v0.3.0

O visual-explainer transforma um pedido em texto — "explica o ciclo da água", "o motor 4 tempos", "o fluxo HTTP", "como funciona uma tecla mecânica" — numa página HTML autocontida e offline, com narração por voz, câmera passo a passo, play/pause, zoom e peças clicáveis. Você não desenha nada: um agente desenhista escreve uma spec declarativa, o builder assa a voz pelo motor (vox-engine) e entrega o HTML pronto, aberto no navegador.

#O que é

O visual-explainer é um agente desenhista para o GitHub Copilot. Você pede uma explicação visual de um assunto e ele entrega uma página HTML autocontida que anima e deixa você interagir com o conceito — abrir, pausar, acelerar, dar zoom e clicar nas peças.

O truque é não desenhar na mão. O agente escreve uma spec declarativa (JSON) descrevendo o que mostrar, e um builder monta o HTML otimizado, inlinando o motor certo. O resultado é um arquivo único, offline, que abre no seu navegador.

i

O agente é instalado no arranque. Depois de instalar o plugin, reinicie o app uma vez para o agente visual-explainer aparecer no seletor.

#Como usar

  1. Instale o plugin

    Pelo marketplace, com o comando da seção Instalar abaixo.

  2. Reinicie o app

    Uma vez, para o agente desenhista ser descoberto no seletor.

  3. Peça uma explicação

    Selecione o agente visual-explainer (ou peça uma explicação visual) e descreva o assunto: "explica visualmente o modelo atômico de Bohr".

  4. Abra a página

    O agente escolhe o motor, escreve a spec, roda o builder e abre a página no navegador. Os arquivos ficam em Desktop\visual-explanations\.

#Dois motores para dois tipos de arte

Nem toda explicação pede o mesmo motor. O builder inlina o certo para o caso, e o agente raramente desenha do zero — um design system de componentes reutilizáveis cresce a cada arte.

  • Motor vxk (leve, próprio) — para as explicações comuns: órbitas, física, fluxo, algoritmos, linhas do tempo. É o caminho padrão.
  • Motor konva (vendorizado) — para grafos e redes: nós arrastáveis e interação mais pesada, quando a explicação é uma estrutura de ligações.
i

Performance vem embutida: ~30 FPS, DPR limitado a 1.5 e pausa automática quando a aba fica oculta. Tudo offline e num único arquivo.

#Narração por voz, câmera e vista explodida

As explicações não são só desenho: elas falam. O builder assa a narração de cada passo pela voz do motor vox-engine e embute o áudio no próprio HTML — a página conta a história enquanto anima.

  • Narração assada — a página nasce com voz; se o motor não estiver disponível, o build falha em vez de entregar mudo.
  • Câmera passo a passo — a câmera navega, dá zoom e foca uma ideia por vez, com auto-enquadramento que preenche a tela.
  • Vista explodida — para aparelhos: o objeto se desmonta em camadas e a câmera explica cada uma. Conceitos abstratos viram placas; objetos físicos ganham a silhueta real da peça.
  • Design system rico — cards com ícone, chips, medidores, conectores com fluxo e um catálogo de componentes que cresce a cada arte.
i

Requer o motor de voz vox-engine instalado (o mesmo do plugin de voz). O áudio é assado no build e fica embutido — a página toca offline.

#Como funciona

  • Pedido — você descreve o assunto para o agente desenhista.
  • Spec — o agente escolhe o motor e escreve uma spec declarativa em JSON, reusando os componentes do kit.
  • Build — o builder monta o HTML autocontido, inlinando o motor e otimizando para rodar leve.
  • Abre — a página abre no navegador, com play/pause, zoom, arrastar e peças clicáveis.

Como a saída é um HTML único e offline, você pode guardar, abrir de novo ou compartilhar a explicação sem depender de nada — nem do agente, nem da rede.

#Instalar

Registre a vitrine uma vez (se ainda não fez):

⌁ copilot plugin marketplace add AllanSantos-DV/copilot-marketplace

Instale o visual-explainer:

⌁ copilot plugin install visual-explainer@copilot-marketplace

Atualize quando quiser (ou todos com --all):

⌁ copilot plugin update visual-explainer

#Perguntas frequentes

As páginas têm áudio?

Sim. Cada explicação nasce narrada: o builder assa a fala de cada passo pela voz do motor vox-engine e embute o áudio no HTML. A página fala ao abrir e roda offline. Requer o vox-engine instalado; sem ele, o build falha em vez de entregar mudo.

Dá para explicar como um aparelho é montado?

Sim, com a vista explodida: o objeto se desmonta em camadas e a câmera explica cada peça, remontando no fim. Conceitos abstratos (como uma pilha de protocolos) viram placas; objetos com forma física (uma tecla, um aparelho) ganham a silhueta real de cada peça.

Preciso saber desenhar ou animar?

Não. Você descreve o assunto em texto; o agente escreve a spec e o builder monta a página. Você recebe o HTML pronto, animado, narrado e interativo.

A página depende de internet para abrir?

Não. O builder inlina o motor escolhido, então a página é um único HTML autocontido que roda offline no seu navegador. Você pode guardar e reabrir quando quiser.

Qual a diferença entre os motores vxk e konva?

O vxk é o motor leve próprio, para explicações comuns (órbitas, física, fluxo, algoritmos, timelines). O konva é para grafos e redes, com nós arrastáveis e interação mais pesada. O agente escolhe o motor pelo tipo de assunto.

Onde ficam as páginas geradas?

Em Desktop\visual-explanations\. Cada explicação vira um arquivo HTML autocontido nessa pasta.

Por que preciso reiniciar o app depois de instalar?

No arranque, o plugin instala o kit e o agente desenhista em ~/.copilot (idempotente, por versão). O reinício é para o app descobrir o agente visual-explainer no seletor.

Onde mora o código do plugin?

O runtime completo (kit, builder, templates e o agente) é vendado nesta vitrine, em plugins/visual-explainer/payload. O boot.mjs instala esse payload em ~/.copilot no primeiro arranque.
