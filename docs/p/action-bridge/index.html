action-bridge

v1.1.0

O action-bridge conecta o Copilot ao app Action (captura e memória de reuniões, 100% local no Windows). Pelo próprio agente você lista reuniões, lê transcrições, busca na memória semântica cruzada, cura decisões e ações e navega o grafo — e o Action se instala sozinho quando ainda não existe.

⌁ copilot plugin install action-bridge@copilot-marketplace
código↗

  • Reuniões sob comando

    Prepara a gravação, lista e abre reuniões e lê a transcrição diarizada — tudo por conversa.

  • Memória que conecta

    Busca semântica em todo o histórico e sugere quais reuniões passadas se ligam à de agora.

  • Instala o Action sozinho

    Se o app não existe, o bootstrap baixa a última release e registra — sem passo manual.

#O que é

O action-bridge é uma ponte MCP entre o agente do Copilot e o app Action. Ele expõe ao agente as ferramentas do Action — reuniões, transcrição, memória, curadoria e grafo — para você operar tudo por linguagem natural, sem sair do terminal.

O Action guarda os dados localmente (áudio, transcrição e memória em DuckDB na sua máquina). Há um modo equipe opcional (Postgres + pgvector) que você liga explicitamente quando quer um acervo compartilhado.

i

Registra um canvas e usa hooks; depois de instalar, reinicie o app uma vez para o painel e os hooks entrarem.

#Como usar

  1. Instale o plugin

    Pelo marketplace (comando na seção Instalar).

  2. Deixe o Action se instalar

    Na primeira tool que precisar dele, o bootstrap baixa e registra o app — você não faz nada.

  3. Prepare uma gravação

    Peça ao agente para preparar a reunião; o app abre pronto e você aperta Start (a captura é sempre uma ação sua).

  4. Depois, cure e busque

    Peça o resumo, extraia decisões e to-dos, e busque na memória cruzada por assuntos.

!

O app Action é Windows-only. A extensão orquestra tudo, mas a captura e a memória vivem no app local.

#O que dá para fazer

  • Reuniões — listar, abrir, atualizar metadados e renomear locutores.
  • Transcrição — ler segmentos diarizados de cada reunião.
  • Memória semântica — indexar e buscar em todo o histórico, com reuniões relacionadas.
  • Curadoria — salvar resumo, decisões e to-dos, materializados no grafo.
  • Grafo — criar entidades e relações e consultar com SQL de leitura.
  • Modo equipe — conectar a um Postgres + pgvector compartilhado (opcional).

Tudo isso pensado com LGPD em mente: minimização, expurgo por retenção e remoção de perfis de voz quando você pede.

#Instalar

Registre a vitrine uma vez (se ainda não fez):

⌁ copilot plugin marketplace add AllanSantos-DV/copilot-marketplace

Instale o action-bridge:

⌁ copilot plugin install action-bridge@copilot-marketplace

Atualize quando quiser (ou todos com --all):

⌁ copilot plugin update action-bridge
!

Este plugin registra um canvas (painel no app). Depois de instalar, reinicie o app uma vez: o canvas-sync espelha a extensão para ~/.copilot/extensions/ e o app a descobre no próximo boot.

#Estrutura

Runtime vendado em plugins/action-bridge/ — vem do repositório de origem, então não se edita à mão aqui. É isto que o copilot plugin install entrega:

⌁ plugins/action-bridge/
  • boot.mjs Bootstrap de SessionStart: baixa/garante o canvas-sync na máquina. 7.3 KB
  • extension.mjs Extensão in-app: hooks e tools da sessão viva do agente. 566 KB
  • hooks.json Hooks de ciclo de vida (SessionStart e afins). 178 B
  • plugin.json Metadados do plugin + marcadores (extensions / hooks). 800 B

#Perguntas frequentes

Meus dados vão para a nuvem?

Não por padrão — o Action é 100% local e a memória fica em DuckDB na sua máquina. O modo equipe (Postgres+pgvector) é opcional e ligado por você.

Como instalo o app Action?

Você não precisa: na primeira vez que uma ferramenta o exige, o plugin baixa a última release e registra o app automaticamente.
